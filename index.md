# 핸즈온 바이브 코딩 워크숍

### AI 시대의 소프트웨어 엔지니어링 혁신

주식회사 로보코
수석 컨설턴트 정도현

---

# 워크숍 개요

핸즈온 바이브 코딩 워크숍은 단순한 코드 생성이 아닌,
**AI를 통한 문서화·테스트·CI/CD·IaC 자동화**를 다루는 실습형 교육입니다.

---

# 핵심 목표

* AI를 통해 개발 효율 극대화
* 자동화된 소프트웨어 엔지니어링 실현
* 코드 품질과 배포 안정성 확보

---

# 로보코에서 제안하는 바이브 코딩

> "단순히 AI가 자연어를 코드로 변환시켜주는 것이 아닌
> 개발자가 더 높은 수준의 소프트웨어 엔지니어링이 가져오는 이점을 활용해서
> 문제 해결에 집중할 수 있도록 돕는 것"

---

# 모던 소프트웨어 엔지니어링

AI와 함께하는 개발은 **구조적 사고**와 **자동화 프로세스**를 기반으로 합니다.

---

# 바이브 코딩과 모던 엔지니어링의 관계

엔지니어링 수준이 낮으면 결과는 랜덤합니다.

→ 자동화된 **가드레일**과 명확한 **프롬프트 엔지니어링**이 필요합니다.

---

# 모던 소프트웨어 엔지니어링의 핵심

* 개발은 **발견과 학습의 연속**
* 개발자는 **학습의 전문가**
* **복잡성 관리**가 생산성과 품질을 결정

---

# 개발자의 핵심 철학

* 반복
* 피드백
* 점진주의
* 실험
* 경험주의

---

# 복잡성 관리의 핵심 개념

* 모듈성
* 응집성
* 관심사 분리
* 추상화
* 느슨한 결합

---

# 효율적 개발을 위한 전략

* 테스트 가능성
* 배포 가능성
* 속도 향상
* 변수 통제
* 지속적 통합/배포

---

# AI가 돕는 엔지니어링

AI는 반복적 작업을 자동화하고,
개발자는 **핵심 의사결정과 설계에 집중**할 수 있습니다.

---

# 로보코의 바이브 코딩 프로세스

1. 요건 정의 (AI 주도 PRD)
2. 계획 및 태스크 분할
3. 태스크 기반 구현 (TDD)
4. 자동화된 CI/CD 수행
5. PR 생성 → 리뷰 → 병합

---

# 1️⃣ 요건 정의

* AI와 함께 PRD 생성
* 기능적/비기능적 요구사항 명확화
* 가드레일 설정

---

# 2️⃣ 계획 및 태스크 분할

* 작업을 명확히 쪼개기
* AI와 함께 WBS 생성
* 각 태스크를 독립적으로 검증 가능하게 설계

---

# 3️⃣ 구현 (TDD 기반)

* 테스트 우선 개발
* AI를 통한 코드 자동화 및 테스트케이스 생성
* 빠른 피드백 루프 확보

---

# 4️⃣ CI/CD 자동화

* GitHub Actions / GitLab CI / AWS CodePipeline 활용
* 테스트, 빌드, 배포 자동화
* AI로 파이프라인 로직 검증

---

# 5️⃣ 코드 리뷰 & 병합

* PR 생성 → AI 기반 품질 분석
* 코드 스타일, 보안 검증
* 팀 기반 리뷰 후 병합

---

# 프롬프트 엔지니어링 핵심

AI를 잘 쓰는 개발자는 **질문을 설계하는 엔지니어**다.

* 컨텍스트 명시
* 기대 결과 정의
* 실패 케이스 사전 설계

---

# 바이브 코딩의 3대 축

1. **명확한 목표 (Goal)**
2. **자동화된 피드백 (Feedback)**
3. **지속 가능한 리듬 (Rhythm)**

---
